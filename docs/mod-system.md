# Mod 系统设计

> 「一个故事结束，另一个故事开始」

## 愿景

## 依赖的 Lua API

Mod 系统依赖引擎提供的 Lua API，详见 [Lua API 设计](./lua-api.md)。

核心依赖：
- `Time` — 时间控制（暂停/恢复/推进）
- `Butterfly` — 蝴蝶效应变量
- `Event` — 事件注册与触发
- `Scene` — 场景加载
- `Entity` — 实体创建

## 剧本创作工作流

### 1. 初始化剧本

```bash
# 使用工具生成模板
./tools/create-mod.sh 我的武侠
# 生成：mods/我的武侠/ 目录结构 + 模板文件
```

### 2. 编写核心脚本

```lua
-- mods/我的武侠/init.lua

-- 注册出身
Origins.register({
  id = "孤儿",
  name = "街头孤儿",
  description = "自幼流落街头，靠偷窃为生",
  stats = {敏捷 = 10, 悟性 = 8},
  start_scene = "扬州/破庙"
})

-- 注册事件链
Events.register("入门考验", {
  trigger = {year = 1, month = 3},  -- 游戏时间第3个月触发
  script = "scripts/入门考验.lua"
})

-- 启动场景
Scene.load("扬州/破庙")
```

### 3. 热重载调试

```lua
-- 开发时自动重载
HotReload.enable()
HotReload.on_reload("scripts/入门考验.lua", function()
  print("入门考验脚本已更新，重新加载事件...")
  Events.reload("入门考验")
end)
```

### 4. 打包发布

```bash
# 打包为 .mod 文件
./tools/pack-mod.sh 我的武侠
# 输出：我的武侠-v1.0.0.mod

# 可上传至：
# - Steam 创意工坊
# - 官方 Mod 站
# - 直接分享文件
```

## 剧本设计指南

### 核心原则

1. **时间驱动**：用时间点触发事件，而非纯任务链
   ```lua
   -- 好：时间到了触发
   Events.register("科举考试", {trigger = {year = 3, month = 8}})
   
   -- 避免：纯任务链
   Quest.chain({"找A", "找B", "找C"})
   ```

2. **蝴蝶效应**：玩家的每个选择都影响后续
   ```lua
   -- 记录关键选择
   Butterfly.set("是否救下神秘人", true)
   
   -- 后续事件检查
   if Butterfly.get("是否救下神秘人") then
     -- 触发报恩剧情
   end
   ```

3. **多结局设计**：终局时根据变量计算结局
   ```lua
   -- 终局计算
   local endings = {}
   if Butterfly.get("声望") > 100 then
     table.insert(endings, "一代宗师")
   end
   if Butterfly.get("恶名") > 50 then
     table.insert(endings, "魔头")
   end
   -- 组合结局：一代宗师·魔头（亦正亦邪）
   ```

## 引擎与剧本的边界

| 引擎负责 | 剧本负责 |
|:---|:---|
| 3D 渲染、物理、动画 | 场景布置、NPC 配置 |
| ECS 系统、实体管理 | 实体行为逻辑、AI |
| Lua 运行时、API 提供 | 脚本编写、事件编排 |
| 存档系统、资源加载 | 存档数据结构定义 |
| UI 框架、输入处理 | UI 布局、交互逻辑 |
| 时间系统、暂停机制 | 时间流速、事件触发点 |
