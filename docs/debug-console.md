# 调试控制台使用指南

> 游戏内实时调试工具，仅在开发模式（`MING_RPG_DEV_MODE=1`）下可用。

---

## 开启与关闭

### 呼出控制台

按 **`~`** 键（ESC 下方的反引号键）打开或关闭调试控制台窗口。

### 环境要求

调试控制台默认在以下模式可用：

| 构建模式 | 状态 |
|:---|:---|
| `debug` | ✅ 可用 |
| `releasedbg`（默认） | ✅ 可用 |
| `release` | ❌ 禁用 |

通过 xmake 构建时，非 release 模式会自动设置 `MING_RPG_DEV_MODE=1`。你也可以手动设置：

```bash
MING_RPG_DEV_MODE=1 cargo run
```

## 控制台命令

在控制台底部的输入框中输入命令，按 **Enter** 执行。

| 命令 | 用法 | 说明 |
|:---|:---|:---|
| `help` | `help` | 显示所有可用命令 |
| `clear` | `clear` | 清空控制台日志 |
| `lua` | `lua <Lua代码>` | 即时执行 Lua 代码 |
| `reload` | `reload` | 热重载 `game/main.lua` |
| `fps` | `fps` | 提示查看右上角性能面板 |
| `entities` | `entities` | 打开 ECS 实体查看器 |
| `editor` | `editor` | 打开场景编辑器 |
| `quit` / `exit` | `quit` | 退出游戏 |

### 命令详解

#### `lua` — 即时执行 Lua

直接在控制台执行任意 Lua 代码，返回值会显示在日志中。

```
> lua print("Hello")
<= Hello

> lua return 1 + 1
<= 2

> lua return Time.get_year()
<= 1573
```

注意事项：
- 代码为空时返回用法提示
- 执行错误会在日志中以红色显示

#### `reload` — 热重载脚本

重新加载 `game/main.lua` 及所有依赖配置，无需重启游戏。

```
> reload
Lua脚本重载成功
```

#### `entities` — 实体查看器

打开一个独立面板，列出当前所有 ECS 实体及其组件信息。用于排查实体生成、组件缺失等问题。

#### `editor` — 场景编辑器

打开可视化场景编辑面板，支持：
- 选择预设类型（Building / Tree / Wall）
- 在场景中点击放置实体
- 撤销最近的操作
- 实时显示放置的实体列表

## 快捷键

| 快捷键 | 功能 |
|:---|:---|
| `~` | 打开/关闭调试控制台 |
| `Alt` | 切换鼠标锁定/释放状态 |

### 鼠标锁定说明

游戏默认锁定鼠标（陀螺仪模式），鼠标移动控制相机旋转。

- **锁定状态**：鼠标不可见，移动控制视角
- **释放状态**：鼠标可见，可点击 UI 或操作系统窗口

按 **Alt**（左或右）在两种状态间切换。

---

## 性能面板

控制台窗口的右上角默认显示实时性能监控：

| 指标 | 说明 |
|:---|:---|
| FPS | 每秒帧数 |
| 帧时间 | 单帧耗时（毫秒） |
| 实体数 | 当前 ECS 实体总数 |
| 三角形数 | 渲染三角形数量 |

点击右上角性能面板的标题栏可以展开/收起详细视图。

## 日志系统

控制台上半部分为日志显示区域，支持以下功能：

### 日志级别筛选

通过工具栏下拉框筛选显示的日志级别：

| 级别 | 颜色 | 说明 |
|:---|:---|:---|
| `Debug` | 灰色 | 调试信息 |
| `Info` | 白色 | 普通信息 |
| `Warn` | 黄色 | 警告 |
| `Error` | 红色 | 错误 |

### 自动滚动

勾选"自动滚动"后，有新日志时自动滚动到底部。取消勾选后可自由浏览历史日志。

### 清除日志

点击"清除"按钮或输入 `clear` 命令清空所有日志。

## 常见问题

**Q: 按 `~` 没有反应？**

A: 检查是否以 release 模式运行。release 模式会禁用所有开发工具。使用 `xmake config -m releasedbg && xmake run` 或手动设置 `MING_RPG_DEV_MODE=1`。

**Q: `lua` 命令执行后没有输出？**

A: `print()` 的输出会直接打印，但如果要看到返回值，需要显式使用 `return`：

```
> lua return Player.get_position()  -- 能看到返回值
> lua Player.get_position()          -- 可能没有输出
```

**Q: 场景编辑器放置的实体能保存吗？**

A: 当前版本为调试工具，放置的实体仅在当前游戏会话中有效，退出后不会保存。后续版本会支持导出为场景配置文件。
